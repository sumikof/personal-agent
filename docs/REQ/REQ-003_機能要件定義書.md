## 2. 機能詳細

### FEAT-001: Redmineタスク作成

**概要**: チャットへの自然言語入力から Redmine にタスク（チケット）を作成する。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-001-01 | ユーザーの自然言語入力を解析し、タイトル・説明・優先度・期日を抽出できること |
| FR-001-02 | 対象 Redmine プロジェクトを指定してチケットを作成できること |
| FR-001-03 | 作成したチケットのURLと概要をチャットに返答できること |
| FR-001-04 | 必須項目（タイトル）が不明な場合、確認メッセージを返せること |

**入力例**:
- 「設計書のレビューについてタスクを作って。期限は来週金曜日。」
- 「明日の14時にミーティングの準備をするタスクを高優先度で登録して」

---

### FEAT-002: Redmineタスク検索・一覧表示

**概要**: 条件を指定してRedmineのタスク一覧を取得・表示する。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-002-01 | ステータス（未着手・進行中・完了等）でフィルタリングして一覧取得できること |
| FR-002-02 | 「今日のタスクは？」などの自然言語での問い合わせに対応できること |
| FR-002-03 | キーワード検索でチケットを絞り込めること |
| FR-002-04 | 取得したタスク一覧をチャットで読みやすい形式で返答できること |
| FR-002-05 | ダッシュボード画面にもリアルタイムで反映されること |

**入力例**:
- 「今日締め切りのタスクを教えて」
- 「進行中のタスクを全部見せて」
- 「○○プロジェクトの未完了タスクは？」

---

### FEAT-003: Redmineタスク更新・進捗報告

**概要**: 既存タスクのステータス・進捗・コメントを更新する。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-003-01 | タスク番号またはタイトルを指定してステータスを更新できること |
| FR-003-02 | 進捗率（0〜100%）を更新できること |
| FR-003-03 | チケットにコメントを追加できること |
| FR-003-04 | 更新対象が曖昧な場合、候補を提示して確認できること |

**入力例**:
- 「チケット#123を完了にして」
- 「設計書レビューのタスクを50%に更新して、コメントに『初回レビュー完了』と書いて」

---

### FEAT-004: タスク優先度・スケジュール調整

**概要**: タスクの優先度変更や期日の調整を行う。AIが未完了タスクを分析し、優先度の提案も行う。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-004-01 | タスクの優先度（緊急・高・通常・低）を変更できること |
| FR-004-02 | タスクの期日（due_date）を変更できること |
| FR-004-03 | 未完了タスクの一覧から、期日・優先度に基づいた対応順序を提案できること |
| FR-004-04 | 担当者（assignee）の変更ができること（セルフアサイン） |

**入力例**:
- 「APIドキュメント作成タスクの優先度を高にして」
- 「今週の優先タスクを整理してレポートして」

---

### FEAT-005: チャットUI

**概要**: エージェントとの対話インターフェース。自然言語でエージェントに指示を出す。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-005-01 | テキスト入力でメッセージを送信できること |
| FR-005-02 | 会話履歴をセッション内で表示・保持できること |
| FR-005-03 | エージェントの応答をMarkdown形式で表示できること |
| FR-005-04 | 送信中はローディング状態を表示できること |
| FR-005-05 | エラー時はわかりやすいエラーメッセージを表示できること |
| FR-005-06 | Enter キーでメッセージ送信できること |

---

### FEAT-006: タスク一覧ダッシュボード

**概要**: Redmine のタスクをリアルタイムで可視化するダッシュボード画面。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-006-01 | 未完了タスクの一覧を表示できること |
| FR-006-02 | タスクをステータス別にグルーピングして表示できること |
| FR-006-03 | 期日が近いタスク（本日・明日）をハイライト表示できること |
| FR-006-04 | 手動またはチャット操作後に自動リフレッシュできること |
| FR-006-05 | タスクのタイトルクリックでRedmineのチケット詳細ページに遷移できること |

---

### FEAT-007: メモ・情報収集（フェーズ2）

**概要**: エージェントがウェブ検索・情報整理を代行する。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-007-01 | キーワードを指定してウェブ検索し、結果を要約できること |
| FR-007-02 | 収集した情報をMarkdown形式でまとめられること |
| FR-007-03 | まとめた情報をファイルまたはメモとして保存できること |

---

### FEAT-008: 予定・カレンダー管理（フェーズ2）

**概要**: スケジュールの確認・登録をエージェントが支援する。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-008-01 | 指定日のスケジュール一覧を確認できること |
| FR-008-02 | 予定を登録・更新・削除できること |
| FR-008-03 | Redmineタスクの期日とカレンダーを連携表示できること |

---

### FEAT-009: 文書作成・編集支援（フェーズ2）

**概要**: レポート・メール等の文書の草案作成をエージェントが支援する。

**機能要件**:

| FR-ID | 要件 |
|-------|------|
| FR-009-01 | テーマ・要点を指定して文書の草案を生成できること |
| FR-009-02 | 生成した草案をMarkdown形式で返答できること |
| FR-009-03 | 既存のテキストを要約・校正・翻訳できること |

## 3. エージェントコア機能要件

### 3.1 LangGraphエージェント

| FR-ID | 要件 |
|-------|------|
| FR-AGT-01 | 自然言語入力を解釈し、適切なツール（MCP等）を呼び出せること |
| FR-AGT-02 | ツール呼び出し結果を自然言語でユーザーに返答できること |
| FR-AGT-03 | 会話コンテキストを保持し、連続した指示に対応できること |
| FR-AGT-04 | エラー発生時は原因をユーザーフレンドリーな言葉で説明できること |
| FR-AGT-05 | ツール呼び出しのログを記録できること |

### 3.2 MCP（Redmine連携）

| FR-ID | 要件 |
|-------|------|
| FR-MCP-01 | RedmineのREST APIをMCPプロトコル経由で呼び出せること |
| FR-MCP-02 | チケットのCRUD操作（作成・取得・更新）をサポートすること |
| FR-MCP-03 | Redmine接続失敗時に適切なエラーを返すこと |
