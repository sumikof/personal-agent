| IF-ID | 接続先 | 通信方式 | 方向 | 認証方式 |

## 3. 外部インターフェース詳細

### IF-001: Anthropic Claude API

| 項目 | 内容 |
|------|------|
| 接続先 URL | `https://api.anthropic.com` |
| 認証 | `x-api-key` ヘッダー（環境変数 `ANTHROPIC_API_KEY`） |
| 主な用途 | 自然言語処理・ツール呼び出し判断・返答生成 |
| 推奨モデル | `claude-opus-4-6` |
| ストリーミング | 使用する（Server-Sent Events） |
| レート制限 | Anthropicのレートリミットポリシーに従う |

**主要API呼び出し**:
```
POST /v1/messages
Content-Type: application/json
x-api-key: {ANTHROPIC_API_KEY}
anthropic-version: 2023-06-01

{
  "model": "claude-opus-4-6",
  "max_tokens": 4096,
  "tools": [...],
  "messages": [...]
}
```

---

### IF-002: Redmine REST API

| 項目 | 内容 |
|------|------|
| 接続先 URL | `http://localhost:8080` （環境変数 `REDMINE_URL`） |
| 認証 | `X-Redmine-API-Key` ヘッダー（環境変数 `REDMINE_API_KEY`） |
| フォーマット | JSON |
| 主な用途 | タスク（チケット）のCRUD操作 |

**主要エンドポイント**:

| エンドポイント | メソッド | 用途 |
|-------------|---------|------|
| `/issues.json` | GET | チケット一覧取得 |
| `/issues.json` | POST | チケット作成 |
| `/issues/{id}.json` | GET | チケット詳細取得 |
| `/issues/{id}.json` | PUT | チケット更新 |
| `/projects.json` | GET | プロジェクト一覧取得 |
| `/issue_statuses.json` | GET | ステータス一覧取得 |

**チケット作成リクエスト例**:
```json
POST /issues.json
{
  "issue": {
    "project_id": 1,
    "subject": "タスクタイトル",
    "description": "タスクの説明",
    "priority_id": 2,
    "due_date": "2026-03-10"
  }
}
```

### IF-003: フロントエンド ↔ バックエンド API

| 項目 | 内容 |
|------|------|
| プロトコル | HTTP（ローカル環境のみ） |
| バックエンド URL | `http://localhost:8000` |
| フォーマット | JSON |
| ストリーミング | Server-Sent Events（SSE）でチャット応答を返す |

**主要エンドポイント**:

| エンドポイント | メソッド | 用途 |
|-------------|---------|------|
| `/api/chat` | POST | チャットメッセージ送信・エージェント応答取得 |
| `/api/tasks` | GET | タスク一覧取得（ダッシュボード用） |
| `/api/health` | GET | サーバー稼働確認 |

**チャットAPIリクエスト例**:
```json
POST /api/chat
{
  "message": "今日のタスクを教えて",
  "session_id": "session-uuid"
}
```

**チャットAPIレスポンス（SSE）**:
```
data: {"type": "token", "content": "今日"}
data: {"type": "token", "content": "のタスクは"}
data: {"type": "tool_call", "tool": "redmine_list_issues", "args": {...}}
data: {"type": "tool_result", "result": {...}}
data: {"type": "done", "content": "完了しました"}
```

## 4. 内部インターフェース

## 5. データフロー

### タスク作成フロー

```
ユーザー
  │ 「タスクを作って」
  ▼
Next.js UI
  │ POST /api/chat
  ▼
Python Backend (FastAPI)
  │ メッセージをLangGraphに渡す
  ▼
LangGraph Agent
  │ Claude APIで自然言語解析
  ▼
Claude API
  │ ツール呼び出し指示（redmine_create_issue）
  ▼
LangGraph Agent
  │ MCPクライアント呼び出し
  ▼
MCP Client
  │ POST /issues.json
  ▼
Redmine
  │ チケット作成完了
  ▼
MCP Client → LangGraph Agent → Claude API → LangGraph Agent
  │ SSEで結果をストリーミング
  ▼
Next.js UI → ユーザーに表示
```
