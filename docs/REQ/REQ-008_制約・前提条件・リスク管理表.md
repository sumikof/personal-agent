## 1. 制約条件

### 1.1 技術的制約

| CON-ID | 制約 | 理由 |
|--------|------|------|
| CON-T-01 | バックエンドは Python 3.11 以上で実装すること | LangGraph・依存ライブラリの要件 |
| CON-T-02 | フロントエンドは Next.js で実装すること | 技術スタック選定（README.md 記載） |
| CON-T-03 | エージェントフレームワークは LangGraph を使用すること | 技術スタック選定 |
| CON-T-04 | タスク管理ツールは Redmine を使用すること | 業務要件 |
| CON-T-05 | Redmine との連携は MCP（Model Context Protocol）経由とすること | 拡張性・標準化のため |
| CON-T-06 | AIモデルは Anthropic Claude API を使用すること | 技術スタック選定 |
| CON-T-07 | 実行環境はローカルPC に限定すること | ユーザー要件 |
| CON-T-08 | Redmine は Docker コンテナで動作させること | ユーザー要件 |

### 1.2 ビジネス制約

| CON-ID | 制約 | 理由 |
|--------|------|------|
| CON-B-01 | シングルユーザー対応のみ実装すること（認証機能不要） | 個人利用のため |
| CON-B-02 | タスクの削除はエージェントを通じて行わないこと | データ保護・誤操作防止 |
| CON-B-03 | クラウドデプロイは対象外とすること | ユーザー要件 |

### 1.3 セキュリティ制約

| CON-ID | 制約 | 理由 |
|--------|------|------|
| CON-S-01 | APIキーはすべて環境変数で管理し、コードに埋め込まないこと | セキュリティ要件 |
| CON-S-02 | `.env` ファイルは `.gitignore` に含めること | 認証情報漏洩防止 |
