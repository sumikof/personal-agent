## 1. ユビキタス言語集

チーム内で統一して使用する用語の定義。開発者・ユーザー・設計書内で以下の用語を一貫して使用する。

| 用語（日本語） | 用語（英語） | 定義 |
|--------------|------------|------|
| エージェント | Agent | LangGraphで構築されたAI処理の主体。ユーザーの指示を受け取り、ツールを呼び出してタスクを実行する。 |
| チャット | Chat | ユーザーとエージェントが自然言語でやり取りするインターフェース。 |
| メッセージ | Message | チャット内でやり取りされる1件のテキスト。ユーザーメッセージ・エージェントメッセージの2種類がある。 |
| タスク | Task | ユーザーが実行すべき作業の単位。Redmineではチケット（Issue）と同義。 |
| チケット | Issue | Redmine上でのタスクの表現形式。ID・タイトル・ステータス・優先度・期日・担当者・説明を持つ。 |
| ステータス | Status | タスクの進捗状態。未着手・進行中・解決・完了・却下 等。 |
| 優先度 | Priority | タスクの重要度・緊急度。緊急・高・通常・低 の4段階。 |
| 期日 | Due Date | タスクの完了予定日。 |
| プロジェクト | Project | Redmine上のタスクのグループ単位。 |
| ダッシュボード | Dashboard | タスクの状況を視覚的に一覧表示する画面。 |
| ツール | Tool | エージェントが呼び出せる機能の単位。MCPサーバーが提供する関数に対応する。 |
| MCP | MCP | Model Context Protocol。エージェントと外部サービス（Redmine等）を繋ぐプロトコル。 |
| MCPサーバー | MCP Server | MCP対応の外部サービスアダプター。Redmine操作を提供するサーバー。 |
| LangGraph | LangGraph | エージェントのワークフロー・状態管理フレームワーク。ノードとエッジで処理フローを定義する。 |
| ノード | Node | LangGraphにおける処理ステップの単位。 |
| 会話履歴 | Conversation History | エージェントが保持するメッセージの履歴。コンテキストとして次の処理に利用される。 |
| ストリーミング | Streaming | エージェントの返答をリアルタイムで逐次表示する方式。 |
| 環境変数 | Environment Variable | システム設定値（APIキー等）を外部から注入するための変数。`.env`ファイルで管理する。 |

## 2. 境界づけられたコンテキスト（Bounded Context）

### 2.1 コンテキストマップ

```
┌─────────────────────────────────────────────────────────────┐
│                     personal-agent システム                   │
│                                                             │
│  ┌─────────────────────┐    ┌─────────────────────────────┐ │
│  │   Chat Context      │    │   Task Management Context   │ │
│  │                     │    │                             │ │
│  │  - Message          │───→│  - Task (= Redmine Issue)   │ │
│  │  - Conversation     │    │  - Project                  │ │
│  │  - UserInput        │    │  - Status                   │ │
│  │  - AgentResponse    │    │  - Priority                 │ │
│  └─────────────────────┘    │  - DueDate                  │ │
│                             └─────────────────────────────┘ │
│  ┌─────────────────────┐    ┌─────────────────────────────┐ │
│  │  Agent Core Context │    │   Integration Context       │ │
│  │                     │    │                             │ │
│  │  - Agent            │───→│  - MCPClient                │ │
│  │  - Tool             │    │  - RedmineAPI               │ │
│  │  - WorkflowGraph    │    │  - ClaudeAPI                │ │
│  │  - AgentState       │    │  - HTTPClient               │ │
│  └─────────────────────┘    └─────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 コンテキスト詳細

#### Chat Context（チャットコンテキスト）
- **責務**: ユーザーとエージェントの対話管理
- **主要エンティティ**: Message, ConversationSession
- **主要操作**: メッセージ送信、会話履歴取得

#### Task Management Context（タスク管理コンテキスト）
- **責務**: Redmineタスクの CRUD・状態管理
- **主要エンティティ**: Task（Issue）, Project, Status, Priority
- **主要操作**: タスク作成・取得・更新

#### Agent Core Context（エージェントコアコンテキスト）
- **責務**: 自然言語処理・ツール選択・ワークフロー実行
- **主要エンティティ**: Agent, Tool, AgentState
- **主要操作**: 入力解析、ツール実行、状態遷移

#### Integration Context（統合コンテキスト）
- **責務**: 外部サービス（Redmine・Claude API）との通信
- **主要エンティティ**: MCPClient, RedmineAPIAdapter
- **主要操作**: API呼び出し、レスポンス変換

## 3. ドメインモデル

### 3.1 主要エンティティ

```
┌─────────────────────┐
│  ConversationSession │
│─────────────────────│
│ + session_id: str   │
│ + messages: List    │
│ + created_at: dt    │
└──────────┬──────────┘
           │ 1..*
┌──────────▼──────────┐
│      Message        │
│─────────────────────│
│ + message_id: str   │
│ + role: Role        │ ── Role: {USER, AGENT, TOOL}
│ + content: str      │
│ + created_at: dt    │
└─────────────────────┘

┌─────────────────────┐
│       Task          │
│─────────────────────│
│ + issue_id: int     │
│ + title: str        │
│ + description: str  │
│ + status: Status    │
│ + priority: Priority│
│ + due_date: date    │
│ + project_id: int   │
│ + assignee_id: int  │
│ + progress: int     │  (0-100%)
│ + created_at: dt    │
│ + updated_at: dt    │
└─────────────────────┘

┌─────────────────────┐
│      Project        │
│─────────────────────│
│ + project_id: int   │
│ + name: str         │
│ + identifier: str   │
└─────────────────────┘
```

### 3.2 値オブジェクト（Value Objects）

| 値オブジェクト | 値の種類 |
|-------------|---------|
| Status | 未着手 / 進行中 / 解決 / 完了 / 却下 |
| Priority | 緊急 / 高 / 通常 / 低 |
| Role | USER / AGENT / TOOL |

## 4. ドメインイベント

| イベント名 | トリガー | 後続処理 |
|----------|---------|---------|
| TaskCreated | タスクが作成された | ダッシュボード更新通知 |
| TaskUpdated | タスクが更新された | ダッシュボード更新通知 |
| MessageReceived | ユーザーメッセージを受信した | エージェント処理開始 |
| AgentResponseGenerated | エージェントが返答を生成した | チャット画面への表示 |
| ToolCalled | エージェントがツールを呼び出した | ログ記録 |
