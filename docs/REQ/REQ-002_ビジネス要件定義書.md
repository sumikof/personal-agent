## 3. 現状分析（As-Is）

### 3.1 現状の業務フロー（タスク管理）

```
1. ブラウザでRedmineを開く
2. プロジェクトを選択する
3. 新規チケット作成画面を開く
4. タイトル・内容・優先度等を手入力する
5. 保存する
```

**課題**:
- 操作ステップが多く手間がかかる
- 作業中に別画面を開く必要がある
- 優先度の整理・確認に一覧を見返す必要がある

### 3.2 現状の業務フロー（情報収集）

```
1. ブラウザで検索する
2. 複数タブで情報を収集する
3. テキストエディタやメモにコピペ・整理する
```

**課題**:
- 情報整理に時間がかかる
- まとめた情報の再利用が困難

## 4. 将来像（To-Be）

### 4.1 目指すべき業務フロー（タスク管理）

```
1. チャット画面で「〇〇についてタスクを作って」と入力する
2. エージェントがRedmineにタスクを登録する
3. 完了メッセージと登録結果を受け取る
```

### 4.2 目指すべき業務フロー（情報収集）

```
1. チャット画面で「〇〇について調べてまとめて」と入力する
2. エージェントが情報を収集・整理して返答する
```

## 6. ビジネスルール

| BR-ID | ルール | 根拠 |
|-------|--------|------|
| BR-01 | エージェントはRedmineの既存プロジェクトに対してのみ操作する | 誤プロジェクト作成防止 |
| BR-02 | タスク削除操作はエージェントが行わない（確認UIから手動操作） | データ保護・誤操作防止 |
| BR-03 | AIへの入力データ（タスク内容等）はAnthropicに送信される前提で利用する | プライバシー認識 |
| BR-04 | エージェントの操作ログは保存する（デバッグ・確認用） | 透明性確保 |

## 7. 成功基準

| 基準 | 測定方法 | 合格ライン |
|------|---------|-----------|
| タスク作成の自然言語解釈精度 | テストケースでの成功率 | 90%以上 |
| エージェント応答時間 | 通常操作（Redmine CRUD） | 5秒以内 |
| フロントエンドの表示速度 | 初回ロード | 3秒以内 |
| システム稼働率 | ローカル環境での起動成功率 | 99%（手動起動時） |
