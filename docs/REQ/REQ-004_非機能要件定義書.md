# REQ-004 非機能要件定義書

| 項目 | 内容 |
|------|------|
| ドキュメントID | REQ-004 |
| プロジェクト名 | personal-agent |
| 作成日 | 2026-03-03 |

---

## 1. 性能要件

### 1.1 応答時間

| NFR-ID | 要件 | 目標値 | 測定方法 |
|--------|------|--------|---------|
| NFR-PERF-01 | チャットへの応答開始時間（ストリーミング開始） | 2秒以内 | APIリクエストから最初のトークン受信まで |
| NFR-PERF-02 | Redmine CRUD操作の完了時間 | 5秒以内 | リクエスト送信から完了レスポンスまで |
| NFR-PERF-03 | ダッシュボードの初期表示時間 | 3秒以内 | ページロードからタスク表示まで |
| NFR-PERF-04 | チャットUIのページ初回ロード | 3秒以内 | ローカル環境での計測 |

### 1.2 スループット

| NFR-ID | 要件 | 備考 |
|--------|------|------|
| NFR-PERF-05 | 同時利用ユーザー数 | 1人（個人利用のため） |
| NFR-PERF-06 | 1分間の最大リクエスト数 | 60回（Redmine API制限考慮） |

---

## 2. 可用性・信頼性要件

| NFR-ID | 要件 | 目標値 |
|--------|------|--------|
| NFR-AVAIL-01 | ローカル環境での起動成功率 | 99%以上（手動起動時） |
| NFR-AVAIL-02 | Redmine接続失敗時のリトライ | 最大3回リトライ後にエラーメッセージ返却 |
| NFR-AVAIL-03 | エージェント処理中の例外ハンドリング | 未処理例外をユーザーに通知し、次の操作を継続できること |

---

## 3. 拡張性要件（最重要）

| NFR-ID | 要件 |
|--------|------|
| NFR-EXT-01 | 新しいエージェントツール（MCP Server）をプラグイン形式で追加できること |
| NFR-EXT-02 | フェーズ2機能（FEAT-007〜FEAT-009）を追加する際にコアアーキテクチャを変更しないこと |
| NFR-EXT-03 | LangGraphのノード・エッジを追加することで新しいワークフローを定義できること |
| NFR-EXT-04 | フロントエンドのコンポーネントをモジュール単位で追加・差し替えできること |

---

## 4. 保守性要件

| NFR-ID | 要件 |
|--------|------|
| NFR-MAINT-01 | エージェントの処理ログをファイルに記録すること（DEBUG/INFO/ERROR レベル対応） |
| NFR-MAINT-02 | 環境変数で接続先・APIキーを管理し、コードに直書きしないこと |
| NFR-MAINT-03 | Python依存ライブラリを `requirements.txt` で管理すること |
| NFR-MAINT-04 | フロントエンド依存を `package.json` で管理すること |

---

## 5. セキュリティ要件

| NFR-ID | 要件 | 優先度 |
|--------|------|--------|
| NFR-SEC-01 | APIキー（Anthropic・Redmine）は環境変数（`.env`）で管理し、バージョン管理に含めないこと | 高 |
| NFR-SEC-02 | `.env` ファイルを `.gitignore` に登録すること | 高 |
| NFR-SEC-03 | Redmine APIへのアクセスは認証トークンを使用すること | 高 |
| NFR-SEC-04 | 個人情報・機密情報をログに平文出力しないこと | 中 |
| NFR-SEC-05 | フロントエンドからバックエンドへの通信はローカルホスト内のみとする | 低（ローカル環境のため） |

---

## 6. 運用・デプロイ要件

| NFR-ID | 要件 |
|--------|------|
| NFR-OPS-01 | `docker-compose up` 相当のコマンド1つでシステム全体（Redmine含む）を起動できること |
| NFR-OPS-02 | README に起動手順・環境変数設定方法を記載すること |
| NFR-OPS-03 | Redmine のデータを永続化できること（Docker Volume 使用） |

---

## 7. ユーザビリティ要件

| NFR-ID | 要件 |
|--------|------|
| NFR-UX-01 | チャット画面はモバイルでも利用可能なレスポンシブデザインであること |
| NFR-UX-02 | 処理中（API呼び出し中）にローディングインジケータを表示すること |
| NFR-UX-03 | エラーメッセージは技術的な内容を排し、対処方法を示した日本語で表示すること |
| NFR-UX-04 | エージェントの思考過程（ツール呼び出し状況）をオプションで表示できること |

---

## 8. 互換性要件

| NFR-ID | 要件 |
|--------|------|
| NFR-COMPAT-01 | Python 3.11 以上で動作すること |
| NFR-COMPAT-02 | Node.js 18 以上で動作すること |
| NFR-COMPAT-03 | モダンブラウザ（Chrome / Firefox / Safari 最新版）で動作すること |
| NFR-COMPAT-04 | macOS / Linux 環境で動作すること（Windows は任意対応） |
