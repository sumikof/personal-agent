## 1. テスト計画の概要

### 1.1 目的

本テスト計画は、personal-agent システムが期待する品質水準を満たすことを確認するための、テスト全体の方針・スコープ・フェーズ・品質基準・スケジュールを定義する。

品質目標:
- ユーザーが自然言語でエージェントに指示し、正確に Redmine タスク操作が実行されることを確認する
- 外部 API（Redmine・Anthropic Claude API）との連携が正常に機能することを確認する
- セキュリティ要件（認証・認可・データ保護）が実装されていることを確認する
- 非機能要件（応答性能・可用性）を満たすことを確認する

### 1.2 テストスコープ

**テスト対象（In Scope）:**
- 認証機能（ログイン・ログアウト・JWT トークン管理）
- エージェント実行機能（自然言語入力・LangGraph ワークフロー・Redmine 操作）
- タスク管理機能（タスク一覧・作成・詳細・更新）
- フロントエンド画面（SCR-001〜008）の UI 動作
- REST API（全エンドポイント）の正常系・異常系
- 外部 IF 連携（Redmine MCP・Anthropic Claude API）
- セキュリティ（OWASP Top 10 基準の脆弱性対策）

**テスト対象外（Out of Scope）:**
- Redmine 自体の機能（外部システムとして扱う）
- Anthropic Claude API の推論精度・モデル品質（外部システムとして扱う）
- インフラ・クラウド環境の負荷試験（別途 OPS フェーズで計画）

## 3. 品質基準

### 3.1 全体品質目標

| 指標 | 目標値 |
|---|---|
| バグ密度（S1: Critical バグ） | リリース時0件 |
| バグ密度（S2: Major バグ） | リリース時0件（要確認） |
| バグ密度（S3/S4: Minor/Trivial バグ） | 5件以下（要確認） |
| テストカバレッジ（バックエンド UT） | ≥ 80%（ライン・ブランチ） |
| テストカバレッジ（フロントエンド UT） | ≥ 70%（要確認） |
| レスポンスタイム（API P95） | ≤ 2秒（通常操作）・エージェント実行は除く（要確認） |
| エージェント実行レスポンスタイム（P95） | ≤ 30秒（要確認：Claude API 依存） |
| 可用性 | ≥ 99.0%（要確認：SLA 要件未確定） |

### 3.2 フェーズ別合格基準

**単体テスト（UT）:**
- コードカバレッジ ≥ 80%（pytest-cov / vitest --coverage で計測）
- 全テストケースがパス（0件のフェイル）
- 重大なロジックバグが存在しないこと

**結合テスト（IT）:**
- 全 API エンドポイント（BST-005 に記載の16エンドポイント）の正常系がパス
- 主要異常系（バリデーションエラー・認証エラー・外部システムエラー）がパス
- 重大バグ（S1/S2）が残存しないこと
- Redmine MCP・Claude API のモックを使ったシナリオテストがパス

**システムテスト（ST）:**
- ユースケーステストシナリオの合格率 ≥ 95%:
  - ログイン〜エージェントでのタスク作成の E2E シナリオ
  - ログイン〜タスク一覧確認〜タスク更新の E2E シナリオ
  - チャット画面でのタスク検索・情報取得シナリオ
- 性能テスト: 同時5ユーザーでの API レスポンス P95 ≤ 2秒（エージェント実行除く）
- セキュリティ診断: OWASP ZAP によるスキャンで High 以上の指摘事項が0件（要確認）

**受入テスト（UAT）:**
- ユーザー（PO）によるテスト完了および承認（要確認：実施の要否・担当者）

## 7. 不具合管理方針

### 7.1 重大度定義

| 重大度 | 定義 |
|---|---|
| S1 (Critical) | サービス停止・データ損失・セキュリティ重大脆弱性レベル |
| S2 (Major) | 主要機能（エージェント実行・タスク管理・認証）が使用不可 |
| S3 (Minor) | 機能は使用できるが動作不正・一部機能の不具合 |
| S4 (Trivial) | 軽微な表示・文言・UX の問題 |

### 7.2 対応方針

- S1: 即時対応（開発継続停止・修正最優先）。Redmine に S1 バグ票を即時作成
- S2: スプリント内対応（現在のスプリントで修正・再テスト実施）
- S3/S4: 次スプリント以降で対応（バックログに積み優先度付けを実施）

### 7.3 不具合管理ツール

- 不具合票の管理: Redmine（本システムのタスク管理基盤を活用）
- 不具合票の属性: タイトル・重大度・再現手順・期待結果・実際の結果・担当者・対象バージョン
